# Character Dossier Workshop

这个 workshop 演示如何把一张角色原始图，逐步扩展成适合游戏、美术提案或品牌世界观展示的完整视觉资产。我们会先使用 GPT-Image-2 生成角色设定资料卡，再基于同一角色继续衍生出游戏 UI 场景，重点观察在多轮生成中如何尽量保持人物识别度、服装设定一致性，以及版式和信息组织的可控性。

使用 GPT-Image-2 的 **edits API（图生图）** 从角色原始图像生成角色设定资料卡和游戏UI界面。

### 优化 Prompt — 强调服装一致性

在 prompt 中明确强调三视图服装一致性和慵懒穿法细节：

```
基于此角色和背景，请制作一份类似官方设定资料的角色资料卡。
• 包含三视图：正面、侧面和背面——三个视图必须保持完全一致的服装穿着状态；
• 重要细节：角色的粉紫色大外套以慵懒方式穿着，一侧滑落至肩膀以下，三个视图都必须保持这一穿法；
• 侧面和背面视图同样必须穿着完整的外套，不能省略；
• 添加角色面部表情的变化，分解并展示服装和装备的详细部分；
• 添加色板，包含世界观设定的简要说明；
• 总体上，使用有组织的布局（白色背景）；
```

### 进一步优化 Prompt — 明确左右肩滑落方向

上一版侧视图有进步，但未区分左右肩的滑落方向。参考图中角色是**右肩滑落**，因此从左侧视图看不应该看到滑落。

```
基于此角色和背景，请制作一份类似官方设定资料的角色资料卡。
• 包含三视图：正面、左侧面和背面——三个视图必须保持完全一致的服装穿着状态；
• 关键服装细节：角色的粉紫色大外套以慵懒方式穿着，右肩一侧滑落至肩膀以下（左肩正常穿着）。因此：
  - 正面视图：右肩外套滑落，左肩正常；
  - 左侧面视图：从左侧看过去，外套在左肩是正常穿着的，不会看到滑落；
  - 背面视图：右肩外套滑落低于左肩，保持不对称穿法；
• 三个视图中外套不能省略或消失，侧面和背面都必须穿着外套；
• 添加角色面部表情的变化，分解并展示服装和装备的详细部分；
• 添加色板，包含世界观设定的简要说明；
• 总体上，使用有组织的布局（白色背景）；
```
